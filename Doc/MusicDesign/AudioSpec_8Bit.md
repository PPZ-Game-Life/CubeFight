# CubeFight - 轻休闲 8-Bit 音频设计规范 (Audio Spec)

**作者**: 继超 (Audio Designer)
**更新日期**: 2026-05-03
**风格定位**: **Light Playful Chiptune (轻快休闲芯片音乐)** —— 保留 8-Bit 识别度，但整体更明亮、松弛、耐听，适合普通休闲游戏长时间停留，不再走紧张压迫路线。

---

## 一、 音乐编曲与 BGM 设计 (Chiptune BGM)

放弃现代交响或绵软的氛围电音，仍以方波、三角波、少量正弦与轻噪声构建 BGM。BPM 控制在 **96-112** 之间，强调轻快推进而不是步步紧逼。

### 1. 主菜单 (Main Menu BGM)
- **情绪基调**: 明亮、友好、进入状态但不压迫。
- **配器构成**: 
  - **Bass**: 更圆润的低频基底，优先用三角波/正弦波，不抢主旋律。
  - **Melody**: 轻跳的三角波或柔和脉冲音，旋律保持短小、亲和。
- **节奏**: 轻微 Swing，但避免强攻击性切分。
- **循环点**: 16 小节无缝 Loop。

### 2. 局内闯关 (In-Game BGM)
- **情绪基调**: 轻松、持续推进、保持玩家操作节奏。
- **配器构成**: 
  - **Percussion**: 低存在感的 kick / clap 组合，避免持续顶耳。
  - **Lead**: 轻亮主音，留出频段给核心动作音效。
- **动态分层 (Adaptive Layering)**:
  - 正常状态：Bass + 轻鼓点 + 稀疏旋律钩子。
  - 当盘面占用率（Fill Ratio）超过 70%，或者触发 3 连击以上时，只做轻微抬升，不再明显“燃起来”。

---

## 二、 核心交互音效设计 (8-Bit SFX)

音效是这个游戏“反馈感”的绝对核心。8-Bit 的优势保留，但现在更强调**柔和、清晰、耐听**，避免高频扎耳。

### 1. 基础 UI 交互 (UI & Menus)
- **Hover / 滑动切片 (Slice)**:
  - **听感**: 短、软、清楚的“咚/滴”。
  - **合成方式**: 以正弦/三角波为主，Duration < 50ms。
  - **频率变化**: 连续滑动切片时只做轻微上扬，不做强烈阶梯感。
- **点击确认 / 开始游戏**:
  - **听感**: 柔和、明确的确认音。
  - **合成方式**: 轻微下滑的三角波/正弦波。

### 2. 核心盘面操作 (Core Gameplay Actions)
这是玩家整场游戏听得最多的声音，必须做到“听一万次也不烦”。

- **选中蓝块 (Select)**:
  - **听感**: 轻微上扬的“啾”或“叮”。
- **蓝块合成 (Merge)**:
  - **听感**: 升级、长大、很顺手。
  - **合成方式**: 上行琶音保留，但更圆、更轻，不做尖锐报警感。
- **蓝块吞噬红块 (Devour Red)**:
  - **听感**: 清脆的“吞掉了”反馈。
  - **合成方式**: 轻噪声 + 低频下坠，保留成功感但减少破坏压迫。
- **蓝块吞噬黄块 (Devour Yellow / Coin)**:
  - **听感**: 亮一点的“叮”或“铃”。
  - **合成方式**: 高音区点缀即可，不再做特别尖的金币声。

### 3. 连击系统 (Combo System)
- **Combo 叠加 (x2, x3, x4...)**:
  - 继续用 Pitch Scaling，但上移幅度收敛，避免越打越紧张。
  - x5 只加一层轻亮泛音，不做刺耳高频尖峰。

## 三、 音频技术与引擎实装规范 (Tech Guidelines)

为了适应 H5/小游戏的严苛包体要求，同时保证多音效并发时不爆音，必须遵守以下混音与实装原则：

### 1. 资产规格与压缩极限
- **格式**: 所有音效和 BGM 统一使用 `.m4a` 或 `.mp3` 格式。如果是极短的音效且需要绝对的无延迟，可考虑无损压缩的 `.wav`（但需严格控制数量）。
- **采样率**: BGM 可降采样至 `22.05kHz`，SFX 可降至 `11.025kHz` 或保留 `44.1kHz`。当前风格更偏柔和，低通与降采样要优先防止毛刺。
- **包体目标**: 游戏全局音频资产总和必须控制在 **2MB** 以内。

### 2. 音频精灵图 (Audio Sprite)
由于 H5 浏览器对多文件并发请求的限制，所有极短的 8-Bit 局内音效（选中、切片、合成、吞吃、金币）必须被打包成**单个音频文件 (Audio Sprite)**。
- 引擎端由 `@主程序 Jeffy` 解析 JSON 配置表，按起止时间 (Start/End) 截取播放。
- 这能彻底消除网络请求带来的音频延迟，保证音画同步的极致打击感。

### 3. 全局混音与防爆音 (Mixing & Ducking)
8-Bit 声音由于谐波丰富，多个音效叠在一起仍可能爆亮或刺耳。
- **音量分级 (Gain Staging)**:
  - BGM: 基础音量 -14dB。
  - UI 点击/切片: 基础音量 -8dB。
  - 核心合成/吞噬: 基础音量 -5dB（优先级最高）。
- **动态避让 (Ducking)**:
  - 当发生高级别“合成”或“连击结束”时，BGM 的音量轻微被压低 2-4dB，随后在 0.5 秒内平滑恢复。
- **并发限制**: 同一帧内，同一种音效（如多个金币同时爆开）最多只允许播放 2-3 个实例，多余的直接被引擎截断或丢弃，防止爆音。
