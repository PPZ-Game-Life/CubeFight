# CubeFight - 8-Bit 风格音频设计规范 (Audio Spec)

**作者**: 继超 (Audio Designer)
**更新日期**: 2026-03-22
**风格定位**: **Cyber-Chiptune (赛博芯片音乐)** —— 在精致的 3D 毛玻璃视觉下，注入硬核、纯粹、极具颗粒感的 8-Bit 灵魂。产生强烈的视听反差与复古街机爽感。

---

## 一、 音乐编曲与 BGM 设计 (Chiptune BGM)

放弃现代交响或绵软的氛围电音，全面采用方波、三角波、锯齿波与白噪声构建 BGM。BPM 控制在 **120-135** 之间，强化“解谜与思考”的紧凑节奏，同时不失摇摆的 Groove。

### 1. 主菜单 (Main Menu BGM)
- **情绪基调**: 蓄势待发、数字空间的唤醒。
- **配器构成**: 
  - **Bass**: 浑厚的三角波 (Triangle Wave) 走 16 分音符的切分 Bassline。
  - **Melody**: 带有快速琶音 (Arpeggio，模拟 8-bit 和弦) 的 25% 脉冲方波 (Pulse Wave)。
- **节奏**: 带有轻微的 Swing，听起来像一台正在自检的超级红白机。
- **循环点**: 16 小节无缝 Loop。

### 2. 局内闯关 (In-Game BGM)
- **情绪基调**: 烧脑、机械齿轮咬合的秩序感。
- **配器构成**: 
  - **Percussion**: 纯粹的白噪声 (White Noise) 捏成的 Kick 和 Snare，短促有力。
  - **Lead**: 50% 方波主音，旋律短促跳跃，留出大量频段给局内的合成/吞噬音效。
- **动态分层 (Adaptive Layering)**:
  - 正常状态：只有 Bass 和 鼓点，保持极简（避免吵闹影响思考）。
  - 当盘面占用率（Fill Ratio）超过 70%，或者触发 3 连击以上时，自动推高 Melody 轨道的音量，让音乐瞬间“燃”起来。

## 二、 核心交互音效设计 (8-Bit SFX)

音效是这个游戏“打击感”的绝对核心。8-Bit 音效的优势是**瞬态极强、频率干净**，能瞬间穿透 BGM 给玩家明确的神经反馈。

### 1. 基础 UI 交互 (UI & Menus)
- **Hover / 滑动切片 (Slice)**:
  - **听感**: 极短的“滴”或“咔”。
  - **合成方式**: 短促的三角波，高频衰减极快，Duration < 50ms。
  - **频率变化**: 当玩家连续滑动切片（Layer 1->2->3）时，音效的 Pitch（音调）应该跟随序号产生阶梯式上升。
- **点击确认 / 开始游戏**:
  - **听感**: 清脆的“哔哩”。
  - **合成方式**: 方波快速 Pitch Drop（音调从高到低快速滑落）。

### 2. 核心盘面操作 (Core Gameplay Actions)
这是玩家整场游戏听得最多的声音，必须做到“听一万次也不烦”。

- **选中蓝块 (Select)**:
  - **听感**: 短促上扬的“啾”。赋予方块被激活的能量感。
- **蓝块合成 (Merge)**:
  - **听感**: 经典的 8-bit “吃蘑菇/升级”音效。
  - **合成方式**: 快速的琶音上行（如 C-E-G-C），合成的等级越高（Lv2 -> Lv9），琶音的速度越快、最后落在的音符 Pitch 越高。
- **蓝块吞噬红块 (Devour Red)**:
  - **听感**: 充满破坏感的“碎裂 / 咬碎”声。
  - **合成方式**: 结合白噪声爆发 (Noise Burst) 与低频三角波的重击。听起来像像素块炸开。
- **蓝块吞噬黄块 (Devour Yellow / Coin)**:
  - **听感**: 极其爽脆、明亮的“叮铃”声。
  - **合成方式**: 纯粹的高频方波，极短的 Decay，复刻经典马里奥吃金币的听感，激发多巴胺。

### 3. 连击系统 (Combo System)
- **Combo 叠加 (x2, x3, x4...)**:
  - 不需要复杂的人声配音，直接用 **音调阶梯 (Pitch Scaling)** 解决。
  - 每次连续吞噬/合成时，播放的基础音效 Pitch 自动向上移调（每多一个 Combo，上移 1 个半音或大二度）。
  - 打到 Combo x5 时，额外触发一层高频的“叮～”泛音闪烁。

## 三、 音频技术与引擎实装规范 (Tech Guidelines)

为了适应 H5/小游戏的严苛包体要求，同时保证多音效并发时不爆音，必须遵守以下混音与实装原则：

### 1. 资产规格与压缩极限
- **格式**: 所有音效和 BGM 统一使用 `.m4a` 或 `.mp3` 格式。如果是极短的音效且需要绝对的无延迟，可考虑无损压缩的 `.wav`（但需严格控制数量）。
- **采样率**: BGM 可降采样至 `22.05kHz`，SFX 视频率需求可降至 `11.025kHz` 或保留 `44.1kHz`。由于 8-Bit 本身就是低保真音色，降采样不仅不会破坏听感，反而会增加复古的“脏”感 (Lo-Fi Grunge)。
- **包体目标**: 游戏全局音频资产总和必须控制在 **2MB** 以内。

### 2. 音频精灵图 (Audio Sprite)
由于 H5 浏览器对多文件并发请求的限制，所有极短的 8-Bit 局内音效（选中、切片、合成、吞吃、金币）必须被打包成**单个音频文件 (Audio Sprite)**。
- 引擎端由 `@主程序 Jeffy` 解析 JSON 配置表，按起止时间 (Start/End) 截取播放。
- 这能彻底消除网络请求带来的音频延迟，保证音画同步的极致打击感。

### 3. 全局混音与防爆音 (Mixing & Ducking)
8-Bit 声音由于方波和锯齿波富含奇数/偶数谐波，多个音效叠在一起极易造成频率打架和数字爆音 (Clipping)。
- **音量分级 (Gain Staging)**:
  - BGM: 基础音量 -12dB。
  - UI 点击/切片: 基础音量 -6dB。
  - 核心合成/吞噬: 基础音量 -3dB（优先级最高）。
- **动态避让 (Ducking)**:
  - 当发生高级别“合成”或“连击结束”时，BGM 的音量必须瞬间被压低 (Ducking) 3-5dB，随后在 0.5 秒内平滑恢复。确保重磅音效的绝对穿透力。
- **并发限制**: 同一帧内，同一种音效（如多个金币同时爆开）最多只允许播放 2-3 个实例，多余的直接被引擎截断或丢弃，防止爆音。
