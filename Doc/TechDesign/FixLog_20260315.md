# CubeFight - 修复日志

**日期**: 2026-03-15  
**修复人**: Jeffy

## 修复内容

### 1. ✅ 俯仰角调整
**问题**: 俯仰角范围不符合预期，无法看到方块底部  
**修复**: 
- 调整俯仰角范围从 15°-85° 改为 75°-105°（平视上下各15度）
- 初始角度从 60° 改为 90°（平视）
- 现在可以向上看15度（俯视）和向下看15度（仰视），能看到方块底部

**代码位置**: `src/core/InputManager.ts`
```typescript
private readonly MIN_POLAR_ANGLE = THREE.MathUtils.degToRad(75);  // 俯视15度
private readonly MAX_POLAR_ANGLE = THREE.MathUtils.degToRad(105); // 仰视15度
private polarAngle: number = Math.PI / 2; // 初始90度（平视）
```

### 2. ✅ 初始方块数量修复
**问题**: 初始只有8个方块，不是完整的3x3x3=27个  
**修复**: 
- 重新设计初始布局，生成完整的27个方块
- 分布在三层：底层9个、中层9个、顶层9个
- 包含红黄蓝三色，等级1-2

**代码位置**: `src/core/Game.ts` - `createInitialCubes()`

### 3. ✅ 黄色方块合成
**问题**: 黄色方块无法被选中和合成  
**修复**: 
- 允许选中黄色方块（之前只能选中蓝色）
- 黄色方块合成逻辑已实现（黄+黄，同等级）

**代码位置**: `src/core/Game.ts`
```typescript
// 可以选中蓝色方块或黄色方块
if (cube.color === CubeColor.BLUE || cube.color === CubeColor.YELLOW) {
  this.selectCube(cube);
}

// 黄块合成逻辑
if (sourceCube.color === CubeColor.YELLOW && targetCube.color === CubeColor.YELLOW &&
    sourceCube.level === targetCube.level) {
  return true;
}
```

### 4. ✅ 切面按钮系统重构
**问题**: 切面按钮不够直观，缺少数字按钮和ALL按钮  
**修复**: 
- 重新设计UI，添加三组切面控制（Y轴层、X轴列、Z轴行）
- 每组包含：0、1、2、ALL 四个按钮
- ALL按钮显示所有方块
- 数字按钮显示对应的切面
- 按钮有激活状态高亮

**UI布局**:
```
右侧中间位置：
┌─────────────┐
│  层 (Y)     │
│ [0][1][2][ALL] │
├─────────────┤
│  列 (X)     │
│ [0][1][2][ALL] │
├─────────────┤
│  行 (Z)     │
│ [0][1][2][ALL] │
└─────────────┘
```

**代码位置**: 
- `index.html` - 新增切面控制UI
- `src/main.ts` - `setupSliceButtons()` 方法

### 5. ✅ 鼠标拖拽修复
**问题**: 不按住鼠标时镜头也会滑动  
**修复**: 
- 鼠标按下时立即设置 `isDragging = true`
- 鼠标抬起时检查移动距离和时间，判断是点击还是拖拽
- 使用 `setTimeout` 延迟重置状态，确保点击事件正确触发

**代码位置**: `src/core/InputManager.ts`
