### 6. ✅ R3F 点击路由修复（蓝块合并 / 蓝吃其他颜色）
**问题**: React Three Fiber 版 `clickCube()` 对所有蓝块点击都优先走 `selectCube()`，导致“第二下点蓝块执行合并”的输入被错误改写为重新选中；同时第二下目标点击的路由没有统一走“已选中状态优先提交动作”的判定链，交互层容易继续回归。  
**修复**:
- 在 `selected` 状态下，先判断是否点击当前已选中方块（用于取消选择）
- 再判断点击目标是否属于当前可见合法目标集，命中则统一走 `commitBoardAction()`
- 只有不属于当前动作提交的点击，才退回到普通蓝块选择逻辑
- 新增点击交互回归测试，覆盖“点蓝块后再点高亮蓝块=合并”和“点蓝块后再点高亮黄块=吞噬”

**代码位置**:
- `src/game/state/gameStore.tsx`
- `src/game/state/gameStore.test.tsx`

### 7. ✅ 方块数字面拦截点击修复（实际点击不触发合并/吞噬）
**问题**: `CubeMesh` 的等级数字使用独立 `mesh` 平面覆盖在方块表面，但点击事件只绑定在盒体 `mesh` 上。玩家实际点击数字面时，事件不会路由到 `clickCube()`，所以会表现为第二下点目标方块没反应，进而看起来“无法合并 / 无法吞噬”。  
**修复**:
- 将点击入口提升到外层 `group`，保证点击盒体或数字面都统一触发同一个 `clickCube()`
- 保留盒体自身 `onClick` + `stopPropagation()`，避免命中盒体时冒泡重复触发
- 新增组件级回归测试，直接验证数字面点击能够触发方块点击处理

**代码位置**:
- `src/render/components/CubeMesh.tsx`
- `src/render/components/CubeMesh.test.tsx`

### 8. ✅ Combo 定时器非法调用修复（合并后直接报错）
**问题**: 浏览器环境里把原生 `setTimeout` / `clearTimeout` 直接塞进 `timers` 对象后，再通过 `timers.setTimeout()` 调用，会让原生方法丢失正确宿主，合并或吞噬后进入 Combo 计时阶段就会抛出 `Illegal invocation`。  
**修复**:
- 默认定时器改成包装函数，统一通过 `globalThis.setTimeout()` / `globalThis.clearTimeout()` 调用浏览器原生 API
- 新增回归测试，模拟必须绑定 `globalThis` 的浏览器计时器，覆盖“执行一次吞噬后正常进入 Combo 调度”链路

**代码位置**:
- `src/game/state/gameStore.tsx`
- `src/game/state/gameStore.test.tsx`
