# CubeFight - 方块状态机与生命周期定义 (Cube State Machine)

**分类**: 核心玩法逻辑 (Gameplay)
**作者**: 樊老师 (主策划)
**更新日期**: 2026-03-14

## 一、 设计定位 (策划与程序/美术的分工边界)
**状态机 (State Machine) 百分之百属于【核心玩法策划】范畴。** 
它决定了玩家在某一瞬间“能点谁”、“点下去会发生什么”。美术只负责给这些已经被策划定义好的“逻辑状态”穿上视觉外衣（加特效、加高亮）。

如果底层状态机没有严密定义，必然导致“方块还在天上飞，玩家又点了一次导致报错”等恶性连环 Bug。

---

## 二、 核心状态枚举与逻辑判定规则

每个独立的方块实体，在 3D 空间中必须且只能处于以下五种状态之一：

### 1. `IDLE` (待机状态)
*   **逻辑定义**: 方块安全停留在 3D 逻辑网格 (gridPos) 中，未被选中，也未处于任何运动序列中。
*   **交互判定**: 
    *   **可被点选**: 点击后进入 `SELECTED` 状态。
    *   **物理碰撞体**: 【开启】（响应射线检测 Raycast）。
*   **视觉表现 (美术侧)**: 正常渲染，执行默认的待机动画（如极微弱的上下浮动或内部核心缓慢自转）。

### 2. `SELECTED` (主动激活状态)
*   **逻辑定义**: 玩家单指点击了该蓝块，准备让它去执行“合并”或“吞噬”操作。
*   **交互判定**: 
    *   如果玩家点击其他空白处，状态退回 `IDLE`。
    *   如果玩家点击符合规则的目标方块，该蓝块进入 `MOVING` 状态。
*   **视觉表现 (美术侧)**: 
    *   自身：播放明显的“呼吸灯”高亮，或出现明显的选中外框。
    *   环境响应：触发周围可交互方块的“引导光环”。

### 3. `TARGETED` (被动瞄准状态)
*   **逻辑定义**: 当玩家选中了一个蓝块（A）后，该蓝块周围的邻居方块（B/C/D）所处的状态。
*   **交互判定**:
    *   **有效目标**: 颜色/等级符合规则，响应玩家第二次点击。若被点击，A 进入 `MOVING` 飞向 B。
    *   **无效目标**: 不符合规则（如高级红块），强行点击无效。
*   **视觉表现 (美术侧)**:
    *   有效：发出绿色光圈，提示“可吃/可合”。
    *   无效：整体变灰暗 (Dimmed)，透明度降至 30%，提示“别点我”。

### 4. `MOVING` (位移脱离状态 - 最危险期)
*   **逻辑定义**: 玩家下达了指令，蓝块脱离原坐标，正在飞向目标格子的过程中（时长约 0.2 - 0.4 秒）。
*   **交互判定**:
    *   **物理碰撞体**: 【强制关闭】。
    *   **输入阻断**: 绝对不可被再次点击，免疫一切交互！
    *   **网格占位**: 在逻辑底层，它已经不属于起点，也未完全落入终点。
*   **视觉表现 (美术侧)**: 播放物理位移补间动画 (Tween)，可带轻微的运动模糊或拖尾残影。

### 5. `MERGING` / `DEVOURING` (清算/特效爆发状态)
*   **逻辑定义**: 方块抵达目标位置，完成数值清算（如 2+2=3，或者蓝吃掉红）。在逻辑层，被消耗的方块实体**已经被回收到对象池或销毁**。
*   **交互判定**: 此阶段纯属表现层，逻辑层该方块已不存在。
*   **视觉表现 (美术侧)**:
    *   `MERGING` (合成)：伴随耀眼的高光和合成音效，生成一个等级 +1 的新方块。
    *   `DEVOURING` (吞噬)：目标方块发生碎裂特效或化为光点被吸收。

```text

  |
  v
 IDLE  <--------+ (取消选中)
  |             |
  | (玩家点击)  |
  v             |
SELECTED -------+ 
  |
  | (点击有效目标)
  v
MOVING
  |
  | (到达目标坐标)
  v
MERGING / DEVOURING (逻辑销毁，播放终结特效)
```
