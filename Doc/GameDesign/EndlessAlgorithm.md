# CubeFight - 无尽模式生成算法文档

**作者**: 樊老师 (主策划)
**更新日期**: 2026-04-08
**版本**: v1.0

## 一、 文档目标

本文档定义首发版无尽模式在 **3x3x3 / 4x4x4 / 5x5x5** 三种战场中的统一生成算法骨架与参数化规则，供 Jeffy 直接实现。

算法目标：
- 保证每局有随机性，但不失控
- 开局简单，随着合成与吃分推进逐步升压
- 防止玩家长期依赖单个蓝块无脑吃分
- 三种尺寸共用一套逻辑，只通过参数体现节奏差异

---

## 二、 核心原则

### 1. 随机但可控
系统主体采用**半保底随机池**，不是纯随机，也不是强脚本导演。

- 主体逻辑：按阶段权重随机生成蓝 / 红 / 黄 / 金黄方块
- 兜底逻辑：当局势触发保底条件时，优先修正生成结果

### 2. 前松后紧
- 前期：保证玩家能起盘、能理解玩法
- 中期：开始形成黄块诱惑与红块压力
- 后期：高压生存与资源取舍并存
- 终局：通过空间压缩逼近 Game Over

### 3. 鼓励多蓝运营
系统必须避免“一只高等级蓝块从头吃到尾”的单解玩法。

### 4. 同骨架，不同节奏
3x3 / 4x4 / 5x5 共用一套流程，仅使用不同参数表。

---

## 三、 适用范围

本算法文档**只覆盖**以下无尽战场：
- 3x3x3
- 4x4x4
- 5x5x5

2x2x2 仅用于前置破冰，不纳入本次主要无尽算法调优目标。

---

## 四、 生成触发时机

每次玩家完成一次**有效操作**后，触发一次补位判定：
- 蓝块合成
- 黄块合成
- 蓝块吃红
- 蓝块吃黄
- 蓝块吃金黄

以下行为**不触发**补位：
- 无效点击
- 旋转视角
- 切层操作
- 打开 UI

---

## 五、 单次补位流程

每次补位必须按以下顺序执行。

### Step 1：收集盘面状态
系统读取以下信息：
- 当前棋盘尺寸
- 当前总分
- 当前最高蓝块等级
- 当前蓝块数量
- 当前空位数量
- 最近 6 次有效得分操作的来源蓝块分布
- 最近若干步里黄块 / 金黄出现间隔

### Step 2：判定难度阶段
根据 **当前总分 + 当前最高蓝块等级** 判定阶段：
- 前期
- 中期
- 后期
- 终局

首发版按尺寸使用以下默认阈值：

### 3x3x3
- 前期：0 ~ 2,000分 或最高蓝块 ≤ 2
- 中期：2,001 ~ 8,000分 或最高蓝块 = 3~4
- 后期：8,001 ~ 20,000分 或最高蓝块 = 5~6
- 终局：20,001分以上 或最高蓝块 ≥ 7

### 4x4x4
- 前期：0 ~ 5,000分 或最高蓝块 ≤ 2
- 中期：5,001 ~ 20,000分 或最高蓝块 = 3~4
- 后期：20,001 ~ 60,000分 或最高蓝块 = 5~6
- 终局：60,001分以上 或最高蓝块 ≥ 7

### 5x5x5
- 前期：0 ~ 10,000分 或最高蓝块 ≤ 2
- 中期：10,001 ~ 40,000分 或最高蓝块 = 3~4
- 后期：40,001 ~ 120,000分 或最高蓝块 = 5~6
- 终局：120,001分以上 或最高蓝块 ≥ 7

实现规则：分数与最高蓝块等级任一满足更高阶段条件时，即提升到对应阶段。

阶段用于决定：
- 颜色生成权重
- 等级范围
- 位置刷点偏好
- 单蓝主导反制强度

### Step 3：执行保底规则
在随机抽取前，先依次检查：

#### A. 蓝块运营保底
如果盘面蓝块过少，或蓝块来源断档，优先提高蓝块生成权重，避免盘面退化成纯垃圾盘。

首发硬规则：
- 3x3 当前蓝块数量 < 2 时，下一次补位蓝块权重至少提升到当前池第一位
- 4x4 / 5x5 当前蓝块数量 < 3 时，下一次补位蓝块权重至少提升到当前池第一位

#### B. 黄系节奏保底
如果黄块 / 金黄长时间未出现，则提高黄系生成权重，保证局内“贪心目标”持续存在。

首发硬规则：
- 连续 5 次补位未出现黄系时，下次补位强制在黄 / 金黄中二选一
- 其中金黄只在该阶段允许出现时参与抽取

#### C. 单蓝主导反制
如果系统检测到某一个蓝块正在持续主导吃分，则：
- 提高其周边红块权重
- 降低其周边黄块 / 金黄权重
- 让高收益目标更偏向生成在远端或其他蓝块可运营区域

### Step 4：决定颜色
如果保底没有直接指定结果，则从以下候选中按权重抽取：
- 蓝
- 红
- 黄（其中可能转化为金黄稀有变体）

阶段性原则：
- 前期：蓝、黄权重更高，红偏低
- 中期：红开始上升，黄稳定
- 后期：红明显占优，黄减少但价值更高
- 终局：红高压，黄 / 金黄作为高风险诱饵存在

### Step 5：决定等级

#### 蓝块等级
- 常规为 1~2 级
- 极少数允许 3 级
- 禁止直接空降高等级蓝块

#### 红块等级
- 开局偏低
- 随总分和最高蓝块等级逐步抬升
- 中后期通常要接近玩家主力蓝块的可吞噬边界，避免被一路吃穿

#### 黄块 / 金黄等级
- 中前期以低中等级为主
- 后期允许更高等级出现
- 金黄必须具备显著诱惑价值
- 金黄不是独立等级体系，而是黄色方块的稀有变体，继承黄色等级、合成与得分规则

### Step 6：决定刷点
生成位置不能纯随机，需要为空位计算位置权重。

位置权重主要考虑：
- 是否靠近当前最强蓝块
- 是否靠近最近连续吃分的蓝块
- 是否位于关键通路
- 是否处于边角或中心
- 是否远离当前主操作区

首发版运行时定义：
- **关键通路**：任意蓝块与其当前可合成对象 / 可吞噬对象之间的最短一步邻接扩展区域
- **主操作区**：最近 4 次有效得分操作所在格子的包围区域
- **其他蓝块发展区**：距离当前主吃分蓝块 ≥ 2 步，且邻近至少 1 个其他蓝块的位置

刷点原则：
- **蓝块**：偏向能形成新支点的位置
- **黄块 / 金黄**：偏向“看得见但不顺手”的位置
- **红块**：偏向关键位、连接位、强蓝附近，制造压力

## 六、 尺寸节奏参数

三种尺寸共用同一流程，但节奏目标不同。

### 1. 3x3x3 - 紧凑生存型
- 前期保护：短
- 红块升压速度：快
- 黄块价值感：中
- 金黄出现感：低
- 单蓝反制速度：快
- 局长目标：短
- 核心体验：生存、快速重开

适合让玩家迅速进入紧张节奏，不允许长时间舒服运营。

### 2. 4x4x4 - 运营拉扯型
- 前期保护：中
- 红块升压速度：中
- 黄块价值感：高
- 金黄出现感：中
- 单蓝反制速度：中
- 局长目标：中
- 核心体验：控局、转线、做支点

这是最适合体现玩法深度的标准无尽战场。

### 3. 5x5x5 - 冲榜长局型
- 前期保护：长
- 红块升压速度：前慢后快
- 黄块价值感：很高
- 金黄出现感：中高
- 单蓝反制速度：中
- 局长目标：中长（仍需控制在 4~8 分钟主目标）
- 核心体验：高分运营、终局冲榜

系统需要允许更长的成长过程，但在中后期通过空间切断和通路压缩快速收口。

## 七、 防止单蓝块无限吃分

### 1. 目标
不硬性锁死最强蓝块，而是防止它长期成为唯一最优解。

### 2. 识别规则：单蓝主导状态
系统维护最近 **6 次有效得分操作**。

这里的“有效得分操作”包含：
- 蓝块合成
- 黄块合成
- 蓝块吃红
- 蓝块吃黄
- 蓝块吃金黄

如果满足以下条件，则判定进入 **单蓝主导状态**：
- 同一个蓝块贡献了 **4 次及以上**
- 且其中至少 **2 次为吞噬得分**
- 且该蓝块当前为盘面最高级或次高级蓝块

### 3. 反制规则

#### 反制 A：强蓝附近提高红块权重
在该蓝块周边空位中，提高红块生成概率，并提高“刚好不能轻松连吃”的红块等级概率。

#### 反制 B：强蓝附近降低黄 / 金黄权重
减少高收益目标在其附近刷新，避免资源自动喂到最强蓝嘴边。

#### 反制 C：高收益目标远端化
黄块与金黄更偏向生成在：
- 离当前主吃分蓝块较远的位置
- 靠近其他蓝块潜在发展区的位置
- 需要转线后才能到达的位置

#### 反制 D：关键位封锁
提高其关键连接位、合成通路和转移路线上的红块刷点权重。

### 4. 解除反制条件
满足以下任一条件，可逐步解除单蓝主导反制：
- 最近 6 次操作中不再由同一个蓝块主导
- 场上出现第二个可运营的中高等级蓝块
- 玩家开始在不同区域交替合成 / 吃分
- 该蓝块连续若干步没有有效得分

首发硬规则：
- 若最近 6 次有效得分操作中，主导蓝块贡献次数下降到 ≤ 3，则反制降为观察态
- 连续 2 次补位未再次进入主导条件，则完全解除反制

### 5. 实现原则
- 不做显式疲劳提示
- 不直接削减得分
- 只通过生成位置、颜色和等级分布改变“顺手度”

## 八、 随机池与保底池的关系

首发版采用 **随机池为主，保底池为辅** 的结构。

### 随机池职责
- 保证每局不重复
- 保证新鲜感

### 保底池职责
- 防止开局无解
- 防止中盘过干
- 防止单蓝吃穿
- 防止高收益目标长期缺席

实现要求：
- 程序层先跑保底判定，再跑随机抽取
- 保底不应频繁硬插手，避免让玩家感到系统强控

## 九、 Jeffy 实现建议

程序层不要写三套分裂逻辑，建议使用：

- 一套统一的 `endlessSpawnPipeline`
- 三套参数配置：
  - `endless_3x3`
  - `endless_4x4`
  - `endless_5x5`

每套配置至少包含：
- 难度阶段阈值
- 颜色生成权重
- 各颜色等级范围
- 单蓝主导反制强度
- 黄 / 金黄保底节奏
- 位置刷点权重

## 十、 验收标准

Jeffy 实现完成后，至少验证以下体验目标：

### 3x3x3
- 玩家能快速起盘
- 不会长时间舒服运营
- 单局明显有“再来一局”感

### 4x4x4
- 玩家能形成中盘运营
- 黄 / 金黄确实构成贪心决策
- 单蓝块不能长期无脑吃穿

### 5x5x5
- 玩家能打出长局高分
- 中后期压力能明显陡增
- 不会演变为单蓝块永久收菜局

通用验收：
- 有随机性
- 开局明显较简单
- 随总分与合成推进而升压
- 多蓝运营优于单蓝吃穿
