# Select SFX Tuning

更新时间：2026-03-25

## 目标

- 目标音效：`select_blue`
- 目标听感：更柔和、更像毛玻璃 UI 被点亮一下的轻脆确认声
- 保留反馈：仍然要有短促上扬的“选中”提示，但避免高频扎耳与连续点击疲劳

## 调整策略

### 频段
- 从偏硬的脉冲波改为更圆一些的三角波，减少上中高频谐波毛刺
- 将主音高上扬区间收窄到 `E5 -> B5`，避免过亮的尖峰感
- 仅保留一层极轻的高频辅助，提供空气感，不再让高频做主角

### 包络
- 轻微放慢攻击段，让瞬态边缘更圆
- 缩短主体停留，保持“短、准、轻”
- 尾部快速释放，避免和后续操作音频打架

### 混音 / 播放
- 运行时播放增益下调：`1.0 -> 0.78`
- 仍维持 `maxInstances: 2`，避免快速改选时叠加过亮
- 优先级继续低于 `merge` / `devour`，不抢核心操作打击感

## 与现有规范对齐

- 对齐 `Doc/MusicDesign/AudioSpec_8Bit.md` 中“选中蓝块”为短促上扬提示的定义
- 对齐 `Doc/TechDesign/AudioPipeline.md` 中 Audio Sprite 与 H5 低包体方案

## 交付备注

- 本次为程序合成原型微调，未改动 cue 名称与时长，避免影响 Sprite 切片索引
- 若后续仍觉得偏亮，可继续从 `playSelect` 增益和辅助泛音层两个维度再收一档
