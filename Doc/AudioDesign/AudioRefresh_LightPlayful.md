# Audio Refresh - Light & Playful 8-Bit

更新时间：2026-04-08

补充更新：2026-05-03

## 本轮目标

- 保留项目的 8-Bit / Chiptune 识别度
- 将整体氛围从“偏机械、偏硬、偏紧张”调整为“轻快、轻松、活泼”
- 降低突兀音效和连续操作时的听觉疲劳
- 保证合并、吞吃、金币获取这些核心动作仍有明确正反馈

## 风格调整

### BGM
- 主菜单 BGM 改为更明亮的大调/流行游戏机语感，减少压迫感
- 局内 BGM 降低鼓噪与硬边，保留节奏驱动但避免持续顶耳
- Melody 层改得更跳跃、更轻巧，避免像报警音或战斗警报

### SFX
- UI 点击音改为更兼容的柔和确认音，适合高频点击
- 选中、切层、Combo 统一收掉尖锐高频与硬瞬态
- 合并、吞吃保留“有收获”的上扬和颗粒反馈，但减少噪声爆点

## 关键音效设计口径

### 按钮点击
- 听感：短、软、清楚，不刺耳
- 原则：优先兼容各种按钮场景，不做过强风格化攻击性瞬态

### 合并
- 听感：升级、成长、正反馈
- 结构：上行琶音 + 轻亮泛音
- 注意：不能尖，不能像报错提示

### 吞红 / 吞黄
- 吞红：从“炸裂破坏”转成“清脆吞掉”的控制感，保留行动成功反馈
- 吞黄：更亮一点，但仍需控制高频，维持奖励感与多巴胺反馈

### Combo
- 听感：连续成功的奖励提示
- 原则：轻闪、上扬、别吵

## 混音原则

- 下修全局 master / music / sfx 增益，避免整体过顶
- BGM 作为氛围底座，不与 SFX 抢前景
- SFX 保留优先级，但核心动作音不应形成刺耳峰值
- Audio Sprite 总体额外低通，减少 8-Bit 谐波在 H5 外放场景下的毛刺感

## 实装备注

- 本轮继续沿用 `public/audio/generated/` 的程序化生成方案
- 未改 Sprite cue 名称与 JSON 结构，保持引擎接线兼容
- 若后续还要更“治愈”一档，优先继续：
  1. 下修 `devour_red` 的噪声层
  2. 缩小 Combo 高频层
  3. 进一步降低局内 base 鼓点存在感

## 2026-05-03 再降紧张度

- BGM 节奏下修：主菜单约 `104 BPM`，局内约 `108 BPM`，从“紧凑芯片音乐”降到普通休闲游戏节奏。
- 音色降硬度：新增 `sine` 振荡器辅助，低频与高频点缀更多使用正弦/三角波，减少脉冲波毛刺。
- 混音下修：运行时 master / music / sfx 增益整体降低，merge / devour 的 ducking 改为轻压，避免动作音制造战斗压迫。
- Sprite 低通继续收窄到更温和区间，优先适配手机外放与 CrazyGames Web 环境。
