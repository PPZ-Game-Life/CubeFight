# CubeFight - 局内系统 UI 与交互设计规范 (In-Game UI/UX)

**作者**: 国盛 (主美术)
**更新日期**: 2026-03-15
**前置参考**: `VisualTarget.md` (R3F 物理渲染 + 微缩景观), `SystemDesign.md` (核心系统与变现)

---

## 1. 整体视觉基调 (Visual Tone)
基于 R3F 的真实物理渲染，局内 UI 必须遵循 **“退让与悬浮” (Yield & Float)** 的原则。
绝对不能让 UI 遮挡精美的 3D 棋盘。我们将采用 **Spatial UI (3D 空间 UI)** 与 **Glassmorphism (毛玻璃拟物化)** 结合的方案，让 UI 仿佛是悬浮在微缩盆景上方的全息投影。

---

## 2. 局内界面布局 (HUD Layout)
采用“极简三段式”布局，大面积留白，视觉焦点 100% 集中在中央的 3x3x3 棋盘。

### A. 顶部信息栏 (Header)
贴顶悬浮，采用极其轻薄的毛玻璃背板（仅在数字下方有微弱的遮罩）。
- **核心排版问题与解决方案 (UI Refinement)**:
  - **问题**: 左侧积分框极长，右侧金币框极短，视觉重心严重失衡。
  - **方案一 (悬浮极简)**: 彻底去掉厚重的半透明底框！数字和极简 Icon (金币/奖杯) 直接悬浮在屏幕左上/右上角，仅在文字背后叠加极其柔和的径向渐变阴影 (Radial Gradient) 以保证可读性，最大化释放顶部空间。
  - **方案二 (一体药丸)**: 将积分与金币合并为一个居中的“药丸状 (Pill-shape)”毛玻璃容器。左侧积分为主视觉（大字号冷色），右侧金币为辅视觉（带 Icon 的暖色），中间以 1px 细线分隔，确保绝对对称。
- **左上角 - 积分 (Score)**: 
  - 呈现：纯白/冷色发光大字号，代表“纯粹的荣誉”。
  - 动效：每次得分，数字进行强烈的**滚动递增 (Rolling)**，并伴随一个 Q 弹的缩放 (Scale Bounce)。
- **中上方 - 目标进度/总分**: 
  - **关卡模式**: 顶部居中放置一个流线型的进度条 (Progress Bar)，显示当前积分与目标积分的比例。
  - **无尽模式**: 直接展示当前获得的巨额数字（使用等宽数字字体，支持数字滚动的动画效果）。
- **左上角/右上角优化细节**: 
  - 积分和金币数值字号需显著大于其说明文字 (“积分”、“金币”)，拉开**视觉层级 (Visual Hierarchy)**。
  - 顶部“药丸 (Pill)”容器宽度不宜拉扯过长，应**根据内部文字长度自适应收缩 (Fit-Content)**，并保持充足的上下留白 (Vertical Padding)。

*(注：为保证游戏节奏的沉浸感与屏幕利用率，局内 **不设置暂停按钮**。切后台等行为由底层逻辑自动处理失焦暂停。)*

### B. 中央游戏区 (Gameplay Area)
绝对干净，没有任何 2D HUD 遮挡。交互信息全部转化为 **Spatial UI (3D 空间 UI)**。
- **摄像机机位与构图 (Camera & Perspective - 解决 2D 扁平感)**: 
  - 目前方块正对镜头导致严重的 2D 纸片感。
  - 必须采用 **带透视视角的等轴视角 (Isometric-like Perspective Angle)**，比如将相机在 X 轴和 Y 轴分别倾斜一定角度（如 Pitch: 45度，Yaw: 45度），确保玩家能同时看到每个 3D 方块的三个面 (顶面、左面、右面)。
  - 方块与方块之间必须拉开极其微小的缝隙并施加环境光遮蔽 (SSAO)，方块上的数字需要带有空间透视形变，严禁贴在屏幕玻璃上死板朝前。
- **Combo / 连击飘字**: 当发生连续消除时，直接在发生消除的 3D 坐标上方弹出带物理深度的 3D 文字（如 "Combo x3!"）。文字带有自发光（Bloom），并会随着镜头的景深（DoF）微微模糊。
- **升级提示**: 方块合成到高等级（如 Lv.6）时，方块顶部浮现短暂的微缩等级徽章或光环。
- **剖面切割 - 分层控制 (Y轴)**:
  - **核心排版问题与解决方案 (UI Refinement)**: 
    - 弱化边框与底板：面板底色透明度降至 10% 以内，边框改为 0.5px（甚至取消），重度依赖 `backdrop-filter: blur(12px)`。
    - 按钮状态极简化：未选中的按钮 (如 0, 1, 2) 仅保留半透明白字，无底框；选中状态 (Active) 采用纯白底板 + 纯黑反色字体。杜绝厚重的圆角线框。
  - **位置**: 悬浮于屏幕 **右侧**，纵向排列（从上往下排列为 `1`、`2`、`3` 等序号按钮，最上方为 `ALL` 恢复全视按钮）。
  - **样式与交互**: 极简数字风格，半透明磨砂质感圆角按钮。选中某层时，对应数字按钮发出强烈的内发光，并在 3D 空间中产生光栅切片效果。
- **剖面切割 - 分列控制 (X/Z轴)**:
  - **位置**: 悬浮于屏幕 **底部中侧**（在道具栏左侧），横向排列（从左往右排列为 `1`、`2`、`3` 等序号按钮，最左侧或最右侧附带 `ALL` 按钮）。
  - **核心排版规范**: 所有卡片内部必须留出至少 30% 的留白 (Padding)，杜绝文字紧贴边框的拥挤感，让 UI 看起来昂贵且松弛。
  - **逻辑联动**: 纯数字序号设计，摒弃左右文字。此处的“列”需根据当前相机的朝向动态映射到 X 轴或 Z 轴，确保玩家点选序号 `1` 时，削掉的必定是屏幕视觉上的最左侧一排。
  - **移动端竖屏适配 (Mobile Vertical Layout Strategy)**: 
    - 手机竖屏时，屏幕底部空间极其宝贵。当前由于面板散乱导致层叠错位，需采用严格的 **“两行堆叠法”** 收拢底部所有交互控件：
    - **Upper Row (上排)**: `[ 列 XZ 控制盘 ]` —— 撑满可用宽度，内部 4 个按钮 (0, 1, 2, ALL) 水平横向均分。
    - **Lower Row (下排)**: `[ 层 Y 控制盘 (左) ]` + `[ 留白间距 ]` + `[ 道具插槽 (右) ]` —— 左右两块面板必须**底边绝对对齐 (Bottom Alignment)**。层 Y 面板采用 2x2 紧凑矩阵排列，右下角留给炸弹/降级锤等核心道具。
    - **按钮一致性**: “ALL” 按钮的体积必须与普通的数字按钮 (0, 1, 2) 保持一致的几何形状（圆形或等宽圆角矩形），绝不可无限制拉伸。
- **视角重置**:
  - **当前试玩版约束**: 不再提供单独的“重置视角”按钮，避免在局内堆叠冗余操作入口。镜头回正由玩家手势自行完成，切片恢复仅保留 `ALL`。

### C. 底部操作/道具栏 (Footer)
紧贴屏幕底部，为玩家保命和变现提供快速入口。

#### 核心排版问题与解决方案 (UI Refinement)
*   **统一高度与对齐**: 底部所有常驻交互卡片必须强制设定一致的高度 (如 `Height: 72px`)，且底边必须绝对水平对齐 (Bottom Alignment)，边距 (Margin-bottom) 保持一致。
*   **降级低频操作**: 将“大厅”按钮从底部核心区移除，降级为一个极简的“主页/返回 Icon”，放置于屏幕左上角。
*   **重构道具卡片 (Prop Slot)**: 道具框内部排版必须居中且层次分明。上方小字 (如“炸弹”)，正中大数字 (如“1”)。若有获取道具的入口 (如看广告的 `+` 号)，绝不能与数字挤在一起，应设计为独立底色的微型角标 (Badge)，悬挂于卡片右上角。

- **左下角 - 危机提示**: 当棋盘容量不足时，左下角（或全局边缘）泛起极其柔和的红色呼吸光晕（Vignette 效果叠加）。
- **右下角 - 道具插槽 (Props & Help)**:
  - **布局**: 2-3 个圆角矩形的毛玻璃卡槽。内部排版强制居中。
  - **内容**: 炸弹 (Bomb)、降级锤 (Hammer)。
  - **IAA 变现 (Help!)**: 如果道具用完，卡槽变成高亮的带 [+] 号的图标，伴随轻微的上下浮动，吸引玩家点击看广告获取道具。

## 3. 字体与排版规范 (Typography)
- **核心字体**: 采用无衬线几何字体（如 *Poppins*, *Montserrat* 或 *SF Pro Display*）。
- **数字字体**: 积分与金币数字必须使用 **等宽字体 (Monospaced)** 或开启 `font-variant-numeric: tabular-nums`，确保数字滚动时宽度不跳动，极具机械秩序感。
- **字重 (Weight)**: 常用 Medium (500) 和 Bold (700)。坚决抛弃细体（在复杂 3D 背景下看不清）和超粗黑体（显得笨重）。

## 4. UI 材质与风格 (Glassmorphism Parameters)
利用 CSS / R3F 后期处理实现顶级的毛玻璃质感：
- **背景色**: `rgba(255, 255, 255, 0.05)` (暗色皮肤下改为黑色透明)。
- **模糊度 (Blur)**: `backdrop-filter: blur(12px) saturate(150%)` (保证透出的 3D 背景色彩仍然饱满)。
- **边框 (Border)**: `1px solid rgba(255, 255, 255, 0.15)` (高光描边，突出玻璃的物理边缘)。
- **阴影 (Drop Shadow)**: `0 8px 32px rgba(0, 0, 0, 0.2)` (柔和的扩散阴影，增加悬浮感)。

## 5. 背景与场景渲染 (Background & Environment)
既然是微缩景观，局内的大背景绝不是死黑或死白。
- **纯色无缝背景 (Seamless Backdrop)**: 采用莫兰迪色系的极简纯色（如灰雾蓝、浅藕粉），没有任何图片贴图。
- **动态光影 (Dynamic Lighting)**:
  - 一盏柔和的主平行光 (Directional Light) 投射出柔和的 PCF 阴影。
  - 随着时间或合成连击，环境光 (Ambient Light) 会发生极其细微的色温变化（如从清晨的冷白向黄昏的暖金过渡）。
- **镜头互动 (Camera Sway)**: 引入轻微的**视差摇晃 (Mouse/Gyro Parallax)**。当手机陀螺仪倾斜或手指拖拽时，相机围绕棋盘中心点（LookAt）产生小幅度偏移，极大地增强了“桌搭盆景”的真实感。
- **后期管线 (Post-processing)**:
  - **暗角 (Vignette)**: 压暗屏幕四周，强制聚焦中央棋盘。
  - **景深 (DoF)**: 虚化棋盘后方的无缝背景和最靠近屏幕的前景粒子。
  - **辉光 (Bloom)**: 限定阈值，仅让高等级方块的核心和 UI 飘字发光，保持画面干净。

## 6. 动态交互与按键特效 (Interaction VFX)
- **按钮点击 (Click Feedback)**: 
  - 拒绝生硬的瞬间缩放。采用阻尼感极强的 Spring 动画（如 `scale: 0.9` 压下，松开时反弹至 `1.05` 再回到 `1.0`）。
  - 点击瞬间，按钮内划过一道倾斜的白色高光带 (Light Sweep)。
- **道具使用 (Prop Usage)**: 
  - 拖拽道具时，屏幕整体背景微微压暗，突出 3D 棋盘上可作用的目标方块。
  - 释放后，道具图标化为一道流光飞向目标位置，触发纯 3D 的粒子爆炸特效。
